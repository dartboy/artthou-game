# Angle Guessing Game: Stack and Recreation Brief


## Detected Tech Stack

- Frontend framework: React, mounted with `createRoot` into `#root`.
- Build style: production-bundled single page app. The bundle references source maps, but the `.map` files are not included locally.
- Styling: Tailwind CSS 3.4.17 utilities, `styled-components` 5.3.11, Emotion/MUI styling runtime, and a custom `Boston-Regular` font reference.
- UI libraries: Material UI icons/components, React Toastify for notifications, and copy-to-clipboard utilities.
- Game/rendering: native SVG generated by React. The central drawing uses two rays, an arc path, and a center dot.
- State: React hooks plus browser `localStorage` under a `guesses` key for daily guess history.
- Date/random logic: Luxon-style date formatting and `seedrandom`-style deterministic RNG. The daily puzzle seed is the current date string, so every player gets the same angle for that day.
- Networking/accounts: Axios calls to `https://auth.teuteuf.fr` for account data, token refresh, and syncing guesses. Local development switches this base URL to `http://localhost:3000`.
- Monetization/analytics: Google Analytics/gtag consent handling, Snigel/Adngin ad slots, consent manager scripts, and Cloudflare beacon.

## Core Game Brief

Build a daily browser game where the player guesses the angle between two SVG lines. Each day, generate two deterministic random angles from the date, draw them from a fixed center point, and compute the clockwise angular difference as the answer. The player enters a numeric guess from 0 to 360 and has 4 attempts. Store each guess with its signed delta from the answer, reveal feedback after guesses, and end the round when the rounded guess matches the rounded answer or when attempts run out. On end, show the answer in a toast and provide shareable result text.

## Recommended Recreation Stack

Use a clean modern stack rather than trying to reverse-engineer the minified build:

- Vite + React + TypeScript for a fast static single page app.
- Tailwind CSS for layout and controls.
- Native SVG React component for the angle diagram.
- `seedrandom` for deterministic daily puzzles.
- `luxon` or `date-fns` for stable date keys like `yyyy-MM-dd`.
- `react-hot-toast` or React Toastify for result/error messages.
- Optional: `lucide-react` or MUI Icons for help, stats, account, and share controls.
- Optional backend: Supabase, Firebase, or a small Node/Express API if you want accounts, leaderboards, or cross-device sync.
- Optional analytics/ads: add only after the core game is working.

## Minimum Build Scope

1. Create the daily puzzle generator from a date key.
2. Render the SVG angle diagram with two lines and an arc.
3. Add number input, guess button, enter-key submit, and validation.
4. Track guesses, attempts, win/loss state, and answer reveal.
5. Persist guesses per date in `localStorage`.
6. Add a small share/result summary and responsive mobile-first layout.

## Notes

The current copy says "3 tries or less" in the page metadata, but the game bundle and visible UI use 4 attempts. For a recreation, choose one rule and make the metadata, help text, and UI match.
